// 定义

TPA 是什么？

一个基于 Python 的 CLI 工具，通过 pip install ti-project-assistant 一键安装。

它的核心工作：读取 SysConfig 图形化配置 → 调用 TI CLI 生成驱动代码 → 组装 CMake 工程 + VSCode 全套配置

🐍 Python · PyPI 🏗️ CMake 自动化 🖥️ Win / Linux 🔌 XDS110 · DAPLink · JLink

// 方案

一行命令，全流程覆盖

TPA 连接 SysConfig 图形化配置 与 CMake 构建生态，把分散的手工步骤自动化

# 1. 安装 TPA（Python ≥ 3.9）
$ pip install ti-project-assistant

# 2. 在含有 .syscfg 的目录下运行
$ mspm0-init

# 3. 在 VSCode 中打开，F5 一键编译+烧录+调试
$ code . && F5
PyPI 一键安装 自动 CMake 工程 SysConfig CLI 集成 VSCode 开箱即用

// 上手

三步上手，立竿见影

①

安装依赖

pip install ti-project-assistant + ARM GCC + CMake + Ninja + MSPM0 SDK + SysConfig + OpenOCD — 跟着教程一步步走，30 分钟内完成

②

配置芯片

打开 SysConfig GUI，像 CubeMX 一样图形化拖拽引脚、配置时钟和外设，保存 .syscfg 文件到项目目录

③

生成项目

终端运行 mspm0-init — TPA 自动读取配置、生成 CMake 工程和 VSCode 全套配置，然后 code . 直接开始编码

之后修改引脚配置？改完 .syscfg 运行 mspm0-init regenerate 即可，无需重建项目

// 结构

现代化项目架构

分层设计 · 自动生成与手写代码隔离 · 递归编译，新建文件自动参与构建

my_project/
  ├── CMakeLists.txt // 顶层构建定义，自动递归链接
  ├── my_project.syscfg // SysConfig 原始配置（可再编辑）
  ├── config/ // ⚠️ 自动生成 - 勿手动编辑
  │   ├── ti_msp_dl_config.c/h // 驱动库配置
  │   ├── device_linker.lds // 链接脚本
  │   └── device.opt // 编译选项
  ├── inc/ // 头文件（app/driver/modules/utils 分层）
  ├── src/ // 源代码 — 你的代码从这里开始 ↓
  │   ├── main.c
  │   └── app/ · driver/ · modules/ · utils/
  ├── .vscode/ // launch.json · tasks.json · settings.json
  └── build/ // 构建输出 (ELF/HEX/BIN)
