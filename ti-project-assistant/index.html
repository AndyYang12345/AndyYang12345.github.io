// 结构

现代化项目架构

my_project/
  ├── CMakeLists.txt // CMake 构建定义
  ├── my_project.syscfg // 原始 SysConfig 配置
  ├── config/ // 自动生成代码（勿手动编辑）
  │   ├── ti_msp_dl_config.c/h
  │   ├── device_linker.lds
  │   └── device.opt
  ├── inc/ // 头文件（分层）
  │   ├── app/ · driver/ · modules/ · utils/
  ├── src/ // 源代码（在此编码）
  │   ├── main.c // ← 你的代码从这里开始
  │   └── app/ · driver/ · modules/ · utils/
  ├── .vscode/ // VSCode 全套配置
  │   ├── launch.json // 调试配置
  │   ├── tasks.json // 构建/烧录任务
  │   └── settings.json // 状态栏按钮
  └── build/ // 构建输出 (ELF/HEX/BIN)

// 跨平台

Windows & Linux 全支持

🪟 Windows

winget 一行安装基础工具链

TI 工具默认路径 C:\ti\

SysConfig CLI: .bat

调试器即插即用

脚本自动适配 Windows 路径

🐧 Linux

apt 一行安装基础工具链

TI 工具默认路径 ~/ti/

SysConfig CLI: .sh

需配置 udev 调试器权限

脚本最初在 Linux 下编写测试

同一套命令：mspm0-init → code . → F5 — 两个平台体验一致
